I am Kin Lane, the API Evangelist...

I took the summer off last year for a project we called Drone Recovery. I began investing in, and flying drones as part of what we were doing, and unexpectedly got interested in the world of drones. Once I returned I began studying the intersection of drones and my career as the API Evangelist, better understanding how companies are using APIs on the drone, on mobile devices, and in the cloud.

I have a client who expressed interest in my thoughts about an API system for drone post-production, and API driven market and ecosystem. I'm looking for projects to help me push forward my storytelling around APIs, so I set out to craft a prototype API to flush out my thoughts around the idea, acting like I'm developing an API, but just do it as an exercise, without all the cost.

This is my Drone API prototype. It is a Github driven prototype where I walk through all the steps I would take to design, develop, manage, and evolve a drone API. I'm looking to establish an evolving, but simple overview of the API, but with all the detail of how I got here right below the surface, so anyone could learn from my story.

Drones Are Interesting

I did not like drones at first. Because of military drones, and Internet of Things hype I just wasn't interested, but over the last year I've come to find them very intriguing. They are the poster child for Internet connected devices, offering the best and worst of what happens when you connect devices to the Internet.

Like much of the API sector, drones fascinate me, but not enough to actually jump into the business, developing devices, tools, APIs, and offering services in the sector. This is what makes my API prototype process great, is that I can explore my ideas, understand the possibilities, and tell stories without any of the financial risk and headach ethat comes along with doing startups.

This prototype is meant to look and act like a Drone API, if I was operating it, has much of the detail about getting started, documentat, code, the APIs, as well as much of the other aspects of operating an API. The entire project is available here on the home page, but to get much of the story I recommend taking a look at the road map, change log, and the blogs, as it tells much of the story of how I explored this idea, with the resulting website as the current snapshot of my drone API prototype.

About My Work

API Evangelist prototypes are how I explore the API lifecycle as it is applied to different industries, and types of digital resources. I'm able to not just able to easily mock an API, I'm able to mock the API operations, while telling the story of it as I flesh out the details of an operation.

These prototypes are part of a new type of storytelling I'm exploring when it comes to helping folks understand the best practices emerging in the API space. If you'd like to invest in storytelling like this, feel free to drop me an email, and I'll see what I can do. I am always looking for new ways to investigate how APIs can be applied when it comes to government, healthcare, financial, and any other industry--I find the process fascinating, and need the work!

Thanks for tuning in. Make sure and come back regularly as this project is living and will continually change as I have new ideas I would like to explore as part of my drone API research, or as I get more investment from a provider, or other company interested in exploring what is possible. Everything here is on Github, so feel free to fork, and run with, or submit an issue of something you want to see.
